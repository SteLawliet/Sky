url-pattern

url-pattern

(必须以/或者*开头)

精确匹配 url-pattern request URL (projectPath)
/FirstServlet http://127.0.0.1:8080/{projectPath}/FirstServlet
/Demo/FirstServlet http://127.0.0.1:8080/{projectPath}/Demo/FirstServlet
模糊匹配 url-pattern request URL(projectPath)
/ http://127.0.0.1:8080/{projectPath}/* (任意一级或多级目录)
/demo/* http://127.0.0.1:8080/{projectPath}/demo/ (demo/下的任意一级或多级目录)
*.do http://127.0.0.1:8080/{projectPath}/ / .do (任意一级或多级目录后以.do结尾的)

  1. *与其他ant风格不同可以代表任意1级或多级目录

  2. / & /*两者一样效果

  3. 前缀模糊匹配和后缀模糊匹配不能同时使用 e.g. /demo// .do

  4. 多个匹配同时时前缀最长者优先.(后缀模糊匹配优先最低)
