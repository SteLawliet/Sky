文件结构

conf

catalina.policy:Tomcat 安全策略文件,控制JVM相关权限,参考java.securityPermission

catalina.porperties: Tomcat Catalina行为控制配置文件,如Common ClassLoader

logging.properties :Tomcat 日志配置文件,JDK logging

server.xml: Tomcat Server配置文件
Server

  • Listener ….==待补充==
    <Listener className="org.apache.catalina.startup.VersionLoggerListener" />

  • GlobalNamingResource:全局JNDI资源

    • Resource 下面是配置权限由后面的tomcat-users.xml提供数据

  • Service “name=Catalina”

    • Connecctor 配置不同协议不同端口的连接

      1. 参考文件tomcat-7.0-doc/cinfig/http.html

      2. 实现类org.apache.catalina.connector.Connector

      3. Executor 线程池的实现类

        org.apache.catalina.core.StandardThreadExecutor

  • Engine
    <Engine name="Catalina" defaultHost="localhost">

  • realm

  • host 虚拟主机

  • Valve 阀门类似filter?tomcat默认配置的是关于日志的

context.xml:全局Context配置

tomcat-user.xml:Tomcat用户权限配置文件 参考(Realm文件实现方式)

web.xml:Servlet标准的web.xml配置文件,Tomcat默认实现的部分配置:

  • org.apache.catalina.servlets.DefaultServlet (静态资源处理的默认servlet)

  • org.apache.jsper.servlet.JapServlet(Jsp文件处理)

lib

Tomcat jar公用类库的存放

ecj-*.jar :eclipse java编译器

jasper.jar:jsp编译器

logs目录

localhost.${date}.log:当Tomcat webapp无法启动时,查看错误日志

catAlina.${date}.log:控制台输出.System.out外置

webapps

简易的app部署

其他方式.

  1. webapp
  2. 修改conf/server.xml
    添加Context元素:

熟悉配置元素参考org.apache.catalina.core.StandardContext setter方法

  1. 通过conf下的文件夹配置独立的context

    热部署配置<context>中reloadable 属性为true.

e.g:engine.name=sky
host.name=stelawliet
context=demo
实际就在conf/sky/stelawliet/下创建demo.xml 文件
如果直接根路径访问应用/,则ROOT.xml文件
